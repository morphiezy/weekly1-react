Versioning

Versioning merupakan suatu proses dimana kita mengatur dan mentracking sebuah perubahan versi dari source code program kita. Version Control System (VCS) merupakan tools yang memungkinkan para developer untuk melakukan versioning source code program kita. Beberapa versi vcs dari dulu hingga sekarang :

  1. Single User
  2. Decentralized
  3. Distributed

Git & Github

Git merupakan salah satu tools version control system yang banyak digunakan developer saat ini untuk berkolaborasi dalam memanajemen source code project.

Github adalah layanan hosting yang dibuat oleh git , dengan github kita dapat hosting source code dari project kita kedalam github , sehingga akan memudahkan kita untuk berkolaborasi bersama dalam mengembangankan software dan versioning dengan git system.

Branching

Branching artinya kita membuat cabang dari repository utama , di cabang baru ini kita dapat melanjutkan pekerjaan dari repository utama seperti :

  • menambahkan fitur
  • memperbaiki bug
  • membuat versi terbaru dari project
  • tanpa mengacaukan source code project utama yang telah ada.

Branch adalah semacam cabang yang membuat commmit dari perubahan source code yang di kerjakan oleh branch tersebut Berikut ini adalah command basic dalam branch :

  • membuat branch git branch [branch_name]
  • menghapus branch git branch -D [branch_name]
  • melihat daftar branch git branch --list
